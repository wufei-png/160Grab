# 91160-cli 功能分析

> 基于 Java 实现的 91160.com (健康160) 医院挂号自动化工具
位于 /home/wufei/github.com/wufei-png/91160-cli
---

## 核心功能

### 1. 用户认证系统

| 功能 | 说明 |
|------|------|
| **登录认证** | 支持手机号+密码登录 91160 账号 |
| **RSA 加密** | 使用 RSA 公钥加密用户名和密码传输 |
| **验证码处理** | 支持图形验证码自动识别 (ddddocr / 斐斐打码) |
| **Token 获取** | 从登录页面解析 token 用于请求验证 |
| **登录重试** | 登录失败自动重试机制 (最大重试次数可配置) |
| **Cookie 管理** | 自动管理登录会话 Cookie，维持登录状态 |

### 2. 号源刷取系统

| 功能 | 说明 |
|------|------|
| **双通道刷号** | 支持两个刷号接口轮询/单独使用 |
| **- 通道1** | 科室排班页接口 (`/guahao/v1/pc/sch/dep`) |
| **- 通道2** | 医生详情页接口 (`/guahao/v1/pc/sch/doctor`) |
| **定时挂号** | 指定时间点自动开抢 (如系统放号时间) |
| **刷号起始日期** | 自定义从哪天开始刷一周内的号 |
| **刷号间隔** | 可配置刷号休眠时间，支持随机范围 (如 3000-8000ms) |
| **多条件筛选** | 支持按医生、星期、上下午、具体时间点筛选 |

### 3. 预约提交系统

| 功能 | 说明 |
|------|------|
| **自动提交预约** | 刷到号后自动填写并提交预约表单 |
| **就诊人选择** | 指定家庭成员进行挂号 |
| **号源验证** | 提交前验证号源有效性 |
| **失败重试** | 同一号源预约失败自动重试 (最多3次) |
| **结果判断** | 通过页面跳转判断预约是否成功 |

### 4. 配置初始化系统 (`init` 命令)

交互式引导用户完成配置:

1. 选择打码平台 (斐斐打码 / ddddocr)
2. 验证打码服务可用性
3. 登录 91160 账号
4. 选择就诊人
5. 选择城市
6. 选择医院
7. 选择科室
8. 选择医生 (支持多选)
9. 选择周几的号 (支持多选)
10. 选择时间段 (上午/下午)
11. 选择具体时间点 (可选)
12. 配置通知服务 (可选)
13. 生成 `config.properties` 配置文件

## 非核心功能

### 1. 代理支持

| 功能 | 说明 |
|------|------|
| **HTTP/SOCKS 代理** | 支持 http 和 socks 两种代理协议 |
| **代理池** | 从文件读取多个代理地址 |
| **代理选择策略** | 支持轮询 (ROUND_ROBIN) 和随机 (RANDOM) 两种模式 |
| **代理验证** | 启动时自动验证代理有效性，剔除无效代理 |

### 2. 消息通知

| 功能 | 说明 |
|------|------|
| **Server酱通知** | 挂号成功后通过 Server酱 推送微信通知 |

### 3. 数据查询接口

| 功能 | 说明 |
|------|------|
| **城市列表** | 获取支持的城市列表 |
| **医院列表** | 根据城市获取医院列表 |
| **科室列表** | 根据医院获取科室列表 |
| **医生列表** | 根据科室获取医生列表 |
| **就诊人列表** | 获取账号下的家庭成员列表 |
| **服务器时间** | 获取服务器当前时间 (用于定时挂号) |

### 4. 验证码服务集成

| 服务 | 说明 |
|------|------|
| **斐斐打码** | 付费打码平台，需配置 PD 账号和密钥 |
| **ddddocr** | 开源 OCR 服务，需自行部署 91160-ocr-server |
| **验证码收集** | 成功识别的验证码会上报用于模型训练 |

### 5. 请求处理增强

| 功能 | 说明 |
|------|------|
| **请求重试** | 网络请求失败自动重试 (最大10次) |
| **请求日志** | 详细的请求/响应日志记录 |
| **Cookie 持久化** | 自动管理和存储 Cookie |

### 6. 部署支持

| 方式 | 说明 |
|------|------|
| **Docker 部署** | 提供官方 Docker 镜像 |
| **JAR 包运行** | 支持 Java 1.8+ 环境直接运行 |
| **配置文件** | 通过 properties 文件管理所有配置 |

## 核心 API 接口

### 登录相关

| 接口 | 方法 | 说明 |
|------|------|------|
| `/login.html` | GET | 获取登录页面和 token |
| `/ajax/common/checkuser.html` | POST | 检测用户名密码 (V1) |
| `/ajax/common/checkuserV3.html` | POST | 检测用户名密码 (V3) |
| `/login.html` | POST | 执行登录 |
| `/checkcode.html` | GET | 获取图形验证码 |

### 数据查询

| 接口 | 方法 | 说明 |
|------|------|------|
| `/ajax/getunitbycity.html` | POST | 获取城市下的医院 |
| `/home/ajaxgetguahaourl.html` | POST | 获取本地医院 |
| `/ajax/getdepbyunit.html` | POST | 获取医院下的科室 |
| `/guahao/v1/pc/sch/dep` | GET | 科室排班信息 (通道1) |
| `/guahao/v1/pc/sch/doctor` | GET | 医生排班信息 (通道2) |
| `/member.html` | GET | 就诊人列表页面 |

### 预约提交

| 接口 | 方法 | 说明 |
|------|------|------|
| `/guahao/ystep1/uid-{}/depid-{}/schid-{}.html` | GET | 预约页面 (获取表单参数) |
| `/guahao/ysubmit.html` | POST | 提交预约 |

## 配置项说明

### 基础配置

```properties
# 账号信息
userName=手机号
password=密码
memberId=就诊人ID

# 医院信息
cityId=城市ID
unitId=医院ID
deptId=科室ID
doctorId=医生ID (可多选: 1001,1002)

# 时间筛选
weeks=1,2,3,4,5,6,7           # 周几 (1=周一, 7=周日)
days=am,pm                     # 上午/下午
hours=08:00-08:30,10:00-10:30  # 具体时间点 (可选)
```

### 刷号配置

```properties
sleepTime=3000                 # 刷号间隔 (支持范围: 3000-8000)
brushStartDate=2024-01-01      # 刷号起始日期
brushChannel=CHANNEL_1         # 刷号通道 (空=双通道轮询)
```

### 定时配置

```properties
enableAppoint=true             # 开启定时挂号
appointTime=2024-01-01 15:00:00  # 定时时间
```

### 代理配置

```properties
enableProxy=true               # 开启代理
proxyFilePath=proxy.txt        # 代理文件路径
proxyMode=ROUND_ROBIN          # 代理模式 (ROUND_ROBIN/RANDOM)
```

### OCR 配置

```properties
ocrPlatform=2                  # 1=斐斐打码, 2=ddddocr
baseUrl=http://127.0.0.1:8000  # ddddocr 服务地址
pdId=xxx                       # 斐斐打码账号
pdKey=xxx                      # 斐斐打码密钥
```

### 通知配置

```properties
sendKey=xxx                    # Server酱 SendKey
```

## Playwright 版本实现建议

### 优先实现 (核心功能)

1. **登录模块** - Playwright 可以更好处理动态页面和反爬
2. **验证码处理** - 可以截图后调用 OCR 服务 优先级 91160-ocr-server>PaddleOCR>百度api 先只支持 91160-ocr-server
3. **号源刷取** - 双通道刷号逻辑
4. **预约提交** - 表单填写和提交

### 可选实现 (非核心功能)

1. 代理支持, 现在已经是模拟浏览器行为，无需代理
2. 消息通知
3. 定时任务

### Playwright 优势

- 更好的反爬虫绑过能力
- 更真实的浏览器行为模拟
- 更简单的页面解析和表单操作
- 内置等待和重试机制
